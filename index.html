1️⃣ Sliding Window

Use this when the problem talks about a contiguous subarray or substring and you want to avoid recomputing things for every range.

1.1 Fixed Sliding Window

When to use: Window size is fixed at k.
Main idea: Add the new right element and remove the old left element as the window moves.
Typical problems: Maximum sum of size k, average of size k, count negatives in every window.
sum += arr[right]
if (right - left + 1 > k) {
  sum -= arr[left]
  left++
}

1.2 Dynamic / Variable Window

When to use: Window size is not fixed; it changes based on a condition.
Main idea: Expand right until condition breaks, then shrink left until valid again.
Typical problems: Longest substring without repeating characters, longest subarray with sum ≤ k.

1.3 At Most K Window

When to use: Problem says at most k distinct, at most k odd numbers, at most k replacements.
Main idea: Maintain a valid window that never exceeds the allowed count.
Typical problems: Longest substring with at most k distinct characters.

1.4 Exactly K Window

When to use: Problem asks for exactly k distinct/countable constraint.
Main idea: Usually solve using atMost(k) - atMost(k - 1).
Typical problems: Subarrays with exactly k distinct integers.
exactly(k) = atMost(k) - atMost(k - 1)

1.5 Minimum Window Pattern

When to use: Find the smallest valid window satisfying some requirement.
Main idea: Expand until valid, then shrink as much as possible while preserving validity.
Typical problems: Minimum window substring.

1.6 Window with Frequency Map

When to use: Window validity depends on character/number frequencies.
Main idea: Use a map or array to track counts inside the current window.
Typical problems: Find anagrams, character replacement, permutation in string.

2️⃣ Two Pointers

Use this when two indices can move in a coordinated way to reduce time from O(n²) to O(n).

2.1 Opposite Direction Pointers

When to use: Usually on sorted arrays or when choosing from both ends.
Main idea: One pointer starts left, the other right; move based on comparison.
Typical problems: Two Sum II, container with most water, trapping decisions from both ends.

2.2 Same Direction Pointers

When to use: Compacting, filtering, removing duplicates, partitioning arrays.
Main idea: One pointer scans, the other marks where to write the next valid item.
Typical problems: Remove duplicates, move zeros, remove element in-place.

2.3 Slow & Fast Pointer

When to use: Linked lists or cycle-like progression problems.
Main idea: One pointer moves 1 step, the other 2 steps.
Typical problems: Detect cycle, find middle of linked list, happy number.

2.4 Partition Pattern

When to use: Reorder items around a pivot or category.
Main idea: Push matching elements to one side and others to the opposite side.
Typical problems: Dutch national flag, quicksort partition.

2.5 Meeting Point Technique

When to use: Need to know whether two traversals eventually meet.
Main idea: If there is a loop or convergence, differently moving pointers will collide.
Typical problems: Linked list cycle entry.

3️⃣ Prefix Sum / Difference Array

Use this when you repeatedly need information about a range, such as a subarray sum or number of occurrences.

3.1 Basic Prefix Sum

When to use: Many range sum queries on an array.
Main idea: Store cumulative sums so any range can be answered in O(1).
Formula: sum(l..r) = prefix[r] - prefix[l - 1]

3.2 Prefix Sum + HashMap

When to use: Need to detect if some earlier prefix makes current range satisfy a target.
Main idea: If current prefix is p, check whether p - target existed before.
Typical problems: Subarray sum equals k, longest subarray with sum k.

3.3 2D Prefix Sum

When to use: Matrix range sum queries.
Main idea: Precompute cumulative area sums to answer submatrix queries in O(1).
Typical problems: Sum of rectangle in matrix.

3.4 Difference Array

When to use: Many range updates, then final reconstruction.
Main idea: Mark only where an update starts and ends, then take prefix sum at the end.
Typical problems: Increment all values in ranges efficiently.

3.5 Prefix XOR

When to use: XOR range queries or XOR identity tricks.
Main idea: Similar to prefix sum, but with XOR operation.
Typical problems: XOR of subarrays, finding odd-frequency properties.

4️⃣ Hashing / Frequency Map

Use hashing when you need fast lookup, counting, deduplication, or grouping.

4.1 Frequency Counting

When to use: Need counts of values or characters.
Main idea: Store occurrences in a map/array.
Typical problems: Majority element, valid anagram, first unique character.

4.2 Complement Lookup

When to use: For each value, you want to know if its matching partner exists.
Main idea: Store seen values so the needed complement can be checked in O(1).
Typical problems: Two Sum.

4.3 Grouping by Signature

When to use: Need to group elements by some normalized key.
Main idea: Compute a signature and map it to a list.
Typical problems: Group anagrams.

4.4 Set-Based Deduplication

When to use: Only presence matters, not count.
Main idea: Use a set to track what has been seen.
Typical problems: Longest consecutive sequence.

5️⃣ Binary Search

Use binary search whenever the answer space is sorted or the condition is monotonic.

5.1 Classic Binary Search

When to use: Search for a target in a sorted array.
Main idea: Compare middle element and discard half the space each step.

5.2 First / Last Occurrence

When to use: Duplicates exist and boundary position matters.
Main idea: Continue searching left or right even after finding target.
Typical problems: Count of occurrences, search range.

5.3 Lower Bound / Upper Bound

When to use: Need first element ≥ x or > x.
Main idea: Search for the first position where a condition becomes true.

5.4 Binary Search on Answer

When to use: Answer is not in the array but can be tested by a monotonic feasibility check.
Main idea: Guess an answer, check if it works, then move left/right.
Typical problems: Koko eating bananas, allocate books, ship packages within D days.

5.5 Rotated Array Search

When to use: Sorted array rotated at some pivot.
Main idea: One half is always sorted; decide whether target lies there.

5.6 Peak Finding

When to use: Need a local maximum/minimum in a structured array.
Main idea: Compare middle with neighbor to know which direction contains a peak.

6️⃣ Sorting-Based Patterns

Sorting is often the transformation step that makes the real pattern easy.

6.1 Interval Merging

When to use: Intervals overlap and need consolidation.
Main idea: Sort by start time, then merge current interval with previous if overlapping.
Typical problems: Merge intervals, insert interval.

6.2 Sweep Line

When to use: Events occur over time or coordinate line.
Main idea: Convert intervals into start/end events, sort events, process running state.
Typical problems: Meeting rooms, skyline ideas, maximum overlap.

6.3 Greedy After Sorting

When to use: Best choice becomes obvious after ordering.
Main idea: Sort, then greedily take valid items.
Typical problems: Activity selection, non-overlapping intervals.

6.4 Custom Comparator Sorting

When to use: Default numeric/alphabetic order is not enough.
Main idea: Define a comparison rule that matches the problem goal.
Typical problems: Largest number, sort by frequency.

7️⃣ Greedy Pattern

Use greedy when a local optimal choice safely leads to a global optimal answer.

7.1 Interval Scheduling

When to use: Need maximum number of non-overlapping intervals.
Main idea: Sort by ending time and keep choosing the earliest finishing valid interval.

7.2 Activity Selection

When to use: Same family as scheduling problems.
Main idea: Earliest finish usually leaves maximum room for later choices.

7.3 Greedy with Heap

When to use: Greedy choice must be dynamically revised among candidates.
Main idea: Push candidates into heap and always extract the currently best one.
Typical problems: Minimum refueling stops, schedule course.

7.4 Greedy with Sorting

When to use: Need an ordering before making local decisions.
Main idea: Sort items, then choose based on current best payoff.

8️⃣ Stack / Monotonic Stack

Use stacks when the problem wants the nearest valid element to the left/right, nesting validation, or controlled reversal of decisions.

8.1 Basic Stack

When to use: Matching, undoing, nested structure validation.
Main idea: Last-in, first-out processing.
Typical problems: Valid parentheses, simplify path.

8.2 Monotonic Increasing Stack

When to use: Need nearest smaller element or increasing structure.
Main idea: Keep stack in increasing order; pop until order restored.
Typical problems: Next smaller element.

8.3 Monotonic Decreasing Stack

When to use: Need nearest greater element.
Main idea: Keep stack in decreasing order.
Typical problems: Next greater element, daily temperatures.

8.4 Histogram / Span Pattern

When to use: Need width expansion controlled by nearest smaller values.
Main idea: Use monotonic stack to find previous and next smaller boundaries.
Typical problems: Largest rectangle in histogram, maximal rectangle.

9️⃣ Queue / Deque / Monotonic Queue

Use queues for level-based processing, and deques when you need window min/max efficiently.

9.1 BFS Queue

When to use: Level-by-level traversal in graphs/trees.
Main idea: Process nodes in the same discovery distance together.
Typical problems: Level order traversal, shortest path in unweighted graph.

9.2 Monotonic Deque

When to use: Need max/min of current window in O(1).
Main idea: Remove weaker candidates from the back; front always holds best answer.
Typical problems: Sliding window maximum.

9.3 Deque Optimization

When to use: Push/pop needed at both ends with order maintenance.
Main idea: Deque acts like a queue + stack hybrid.

🔟 Linked List Patterns

Linked list problems mostly revolve around pointer manipulation and edge-case safety.

10.1 Reverse Linked List

When to use: Need reversed order of nodes.
Main idea: Redirect next pointers one by one.

10.2 Fast & Slow Pointer

When to use: Need middle or cycle detection.
Main idea: Different pointer speeds reveal structure.

10.3 Merge Lists

When to use: Combine sorted linked lists.
Main idea: Repeatedly attach smaller current node.

10.4 Dummy Node Pattern

When to use: Head insertion/deletion causes many corner cases.
Main idea: Add a fake starting node to simplify pointer logic.

10.5 Reverse in Groups / Segments

When to use: Reverse every k nodes or a partial segment.
Main idea: Cut, reverse locally, reconnect carefully.

1️⃣1️⃣ Tree Traversal

Traversal tells you the order in which the tree should be processed.

11.1 DFS Traversals

Preorder: Process node before children.
Inorder: Left → node → right; useful for BST sorted order.
Postorder: Process children before node; useful when answer depends on children.

11.2 BFS / Level Order

When to use: Problem asks level by level, minimum depth, nearest node, etc.
Main idea: Use queue and process one level at a time.

11.3 Zigzag Traversal

When to use: Need alternating direction at each level.
Main idea: BFS plus reverse/order control per level.

11.4 Vertical / Boundary Views

When to use: Problem groups nodes by geometric relation.
Main idea: Track horizontal distance or boundary rules.

1️⃣2️⃣ Tree DFS Problem Patterns

These are not just traversals — they are recurring DFS-style ways of thinking on trees.

12.1 Path-Based DFS

When to use: Need root-to-leaf or any path property.
Main idea: Carry current path sum/string/vector through recursion.
Typical problems: Path sum, binary tree paths.

12.2 Height / Depth Calculation

When to use: Tree depth, balance, longest root-based chain.
Main idea: Return height from child, combine at parent.

12.3 Diameter Pattern

When to use: Need longest path through a node.
Main idea: Use left height + right height while computing heights.

12.4 Subtree Aggregation

When to use: Node answer depends on combined child information.
Main idea: Return subtree sum/count/state upward.
Typical problems: Sum tree, count good nodes, subtree averages.

12.5 Backtracking on Tree

When to use: Need to choose and undo state while moving through branches.
Main idea: Add state before recursion, remove state after recursion.

1️⃣3️⃣ Graph Patterns

Graph questions often reduce to traversal, connectivity, ordering, or component detection.

13.1 Graph DFS

When to use: Explore connected structure deeply.
Main idea: Visit node, recursively visit unvisited neighbors.
Typical problems: Number of islands, connected components.

13.2 Graph BFS

When to use: Need shortest steps in unweighted graph.
Main idea: Traverse by layers of distance.
Typical problems: Word ladder, shortest path in binary matrix.

13.3 Multi-Source BFS

When to use: Multiple starting points spread influence simultaneously.
Main idea: Push all starting sources into queue first.
Typical problems: Rotten oranges, nearest zero in matrix.

13.4 Topological Sort

When to use: Directed acyclic graph with dependency ordering.
Main idea: Use indegree + queue or DFS finishing order.
Typical problems: Course schedule II, alien dictionary.

13.5 Cycle Detection

When to use: Need to know whether a loop exists.
Main idea: Undirected: parent-aware DFS/BFS. Directed: recursion stack or indegree logic.

1️⃣4️⃣ Shortest Path Patterns

Pick the algorithm based on edge weights and graph size.

14.1 Dijkstra

When to use: Weighted graph with non-negative edges.
Main idea: Repeatedly finalize the currently shortest reachable node using a min-heap.

14.2 Bellman-Ford

When to use: Negative edges may exist.
Main idea: Relax all edges up to V-1 times; can detect negative cycles.

14.3 Floyd-Warshall

When to use: Need all-pairs shortest path on small dense graph.
Main idea: Try every node as an intermediate connector.

14.4 0-1 BFS

When to use: Edge weights are only 0 or 1.
Main idea: Use deque; weight 0 edges go to front, weight 1 edges go to back.

1️⃣5️⃣ Union Find / Disjoint Set

Use DSU when the question is about connectivity, merging groups, or checking whether two nodes belong to the same component.

15.1 Basic Connectivity

When to use: Need fast merge and same-group query.
Main idea: Each group has a representative parent.

15.2 Path Compression

When to use: Optimization inside DSU.
Main idea: During find, attach nodes directly to root for future speed.

15.3 Union by Rank / Size

When to use: Optimization inside DSU.
Main idea: Attach smaller tree under larger one to keep depth low.

15.4 Dynamic Connectivity

When to use: Edges are added gradually and connectivity changes over time.
Main idea: Union on every new connection, query whenever needed.

1️⃣6️⃣ Backtracking

Use this when you must explore choices, recurse into each, then undo and try the next one.

16.1 Subsets Pattern

When to use: Every element can be taken or skipped.
Main idea: Binary choice at each index.

16.2 Combinations Pattern

When to use: Need to choose k elements or combinations without order.
Main idea: Choose next candidate starting from current index.

16.3 Permutations Pattern

When to use: Order matters.
Main idea: Try every unused element in current position.

16.4 Constraint Placement

When to use: Items must be placed while satisfying rules.
Main idea: Place only if valid, recurse, then remove.
Typical problems: N-Queens, sudoku solver.

16.5 DFS on Board with Undo

When to use: Grid search where each path has temporary visited state.
Main idea: Mark visited, recurse neighbors, unmark on return.
Typical problems: Word search.

1️⃣7️⃣ Dynamic Programming

Use DP when the problem has overlapping subproblems and optimal substructure.

17.1 1D DP

When to use: State depends on one index and a few previous states.
Main idea: Build answer left to right.
Typical problems: Fibonacci, climbing stairs, house robber.

17.2 2D DP

When to use: State depends on two dimensions like row/col or i/j.
Main idea: Fill DP table based on smaller states.
Typical problems: Unique paths, LCS.

17.3 Knapsack Pattern

When to use: Choose/pick items under capacity or target constraint.
Main idea: For each item, pick or skip.
Typical problems: 0/1 knapsack, partition equal subset sum, target sum.

17.4 Longest Subsequence DP

When to use: Need longest valid ordered structure, not necessarily contiguous.
Main idea: State often means best answer ending at position i.
Typical problems: LIS, LCS.

17.5 Interval DP

When to use: Problem is naturally defined on subarray ranges.
Main idea: Compute answer for smaller intervals, then expand.
Typical problems: Matrix chain multiplication, burst balloons, palindrome partitioning.

17.6 DP on Trees

When to use: Each node has decision states that depend on children.
Main idea: Return multiple values from child to parent.
Typical problems: House robber III.

17.7 Bitmask DP

When to use: State is a subset and n is small (often ≤ 20).
Main idea: Use bits to represent selected/unselected items.
Typical problems: TSP variants, assignment problems.

1️⃣8️⃣ Heap / Priority Queue

Use heaps when you repeatedly need the smallest/largest/current best element.

18.1 Top K Elements

When to use: Need k best items instead of full sorting.
Main idea: Maintain heap of size k.
Typical problems: Top k frequent elements.

18.2 Kth Smallest / Largest

When to use: Need one ranked element.
Main idea: Use min-heap or max-heap based on viewpoint.

18.3 Merge K Sorted Structures

When to use: Many sorted lists/arrays must be merged.
Main idea: Heap stores current smallest candidate from each source.
Typical problems: Merge k sorted lists, smallest range covering elements from k lists.

18.4 Two-Heap Partition

When to use: Need median or balanced split of data stream.
Main idea: Max-heap keeps smaller half, min-heap keeps larger half.
Typical problems: Find median from data stream.

18.5 Priority Scheduling

When to use: Current next best job/event must be picked repeatedly.
Main idea: Heap orders tasks by priority, deadline, or start time.
Typical problems: Meeting rooms II, task scheduler variants.
